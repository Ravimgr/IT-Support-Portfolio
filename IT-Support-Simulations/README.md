## 🛠️ Common Scenarios Practiced

### 1. Password Reset & Account Unlock

**Scenario:** User locked out of account after multiple failed login attempts.

**Typical Approach:**
1. Verify user identity (ask for employee ID, department)
2. Check account status in system
3. Unlock account and/or reset password
4. Provide temporary password
5. Instruct user to change password at next login
6. Document in ticket system

**Average Resolution Time:** 3 minutes  
**Customer Satisfaction:** 4.7/5

---

### 2. Email Configuration Issues

**Scenario:** User cannot send or receive emails on new device.

**Systematic Approach:**
1. Gather information:
   - Can receive but not send? (SMTP issue)
   - Can send but not receive? (POP3/IMAP issue)
   - Neither? (Account or connectivity issue)
2. Verify email account credentials
3. Check server settings:
   - Incoming: mail.company.com, port 993 (IMAP)
   - Outgoing: smtp.company.com, port 587 (TLS)
4. Test email functionality
5. Verify on multiple devices if needed

**Average Resolution Time:** 8 minutes  
**Customer Satisfaction:** 4.5/5

**Key Learning:** Always check both incoming AND outgoing server settings.

---

### 3. Network Connectivity Troubleshooting

**Scenario:** User cannot access internet or internal resources.

**Diagnostic Steps:**
1. Verify physical connection (cable, WiFi)
2. Check IP configuration:
```
   ipconfig /all
   - Valid IP? (not 169.254.x.x APIPA)
   - Correct subnet mask?
   - Default gateway present?
   - DNS servers configured?
```
3. Test connectivity:
```
   ping 8.8.8.8 (internet)
   ping gateway IP (local network)
   ping company.com (DNS resolution)
```
4. Release/renew IP if needed:
```
   ipconfig /release
   ipconfig /renew
   ipconfig /flushdns
```
5. Check firewall settings
6. Verify network adapter driver

**Average Resolution Time:** 12 minutes  
**Customer Satisfaction:** 4.3/5

**Common Issues Found:**
- DHCP not functioning (APIPA address)
- Incorrect DNS settings
- Disabled network adapter
- WiFi connected to wrong network

---

### 4. Software Installation & Configuration

**Scenario:** User needs software installed or is experiencing application errors.

**Process:**
1. Verify user has appropriate license
2. Check system requirements
3. Uninstall previous version if needed
4. Download from approved source
5. Install with appropriate permissions
6. Configure initial settings
7. Test functionality
8. Provide basic user training if needed

**Average Resolution Time:** 10 minutes  
**Customer Satisfaction:** 4.6/5

---

### 5. Printer Troubleshooting

**Scenario:** Printer not responding or printing incorrectly.

**Troubleshooting Checklist:**
- [ ] Printer powered on and connected?
- [ ] Default printer set correctly in Windows?
- [ ] Print queue cleared? (any stuck jobs?)
- [ ] Printer driver installed and up to date?
- [ ] Network printer: can ping printer IP?
- [ ] Test with different document
- [ ] Restart print spooler service if needed

**Average Resolution Time:** 7 minutes  
**Customer Satisfaction:** 4.4/5

**Common Fixes:**
- Clear print queue
- Restart print spooler service
- Reinstall printer driver
- Set as default printer

---

### 6. VPN & Remote Access Issues

**Scenario:** User cannot connect to company VPN from home.

**Systematic Approach:**
1. Verify VPN client installed and updated
2. Check credentials (username, password)
3. Verify internet connectivity (can browse without VPN?)
4. Check VPN server address correct
5. Verify firewall not blocking VPN ports
6. Check if corporate network blocking connection
7. Try alternate VPN protocol if available
8. Review VPN logs for error codes

**Average Resolution Time:** 15 minutes  
**Customer Satisfaction:** 4.2/5

---

### 7. Mobile Device Support

**Scenario:** Configure corporate email on mobile device.

**Steps:**
1. Verify device compatibility
2. Obtain user's email credentials
3. Add account to device (Exchange or IMAP)
4. Configure server settings:
   - Server: outlook.office365.com (for O365)
   - Username: user@company.com
   - SSL required
5. Test send and receive
6. Configure security policies if required (PIN, remote wipe)
7. Verify calendar and contacts syncing

**Average Resolution Time:** 12 minutes  
**Customer Satisfaction:** 4.6/5

## 💡 Key Learnings & Insights

### Customer Service Skills

#### What Works Well ✅
- **Active listening:** Let users fully explain issue before jumping to solutions
- **Empathy:** Acknowledge frustration ("I understand this is preventing you from working")
- **Clear communication:** Avoid jargon, explain in user-friendly terms
- **Setting expectations:** "This should take about 10 minutes to resolve"
- **Follow-up:** "Is there anything else I can help you with?"

#### Common Mistakes to Avoid ❌
- Interrupting user while they're explaining
- Using technical jargon without explanation
- Making assumptions without asking questions
- Not verifying the fix worked before closing ticket
- Failing to document solution for future reference

### Technical Troubleshooting

#### Systematic Approach
```
1. LISTEN: Understand the problem fully
2. ASK: Gather information (when, what changed, error messages)
3. ISOLATE: Narrow down to specific component
4. TEST: Verify hypothesis
5. FIX: Implement solution
6. VERIFY: Confirm issue resolved
7. DOCUMENT: Record solution in ticket
```

#### Questions to Always Ask
- "When did this start happening?"
- "What changed recently?"
- "Is anyone else experiencing this?"
- "What error messages do you see?"
- "Can you show me exactly what happens?"

### Time Management

#### Prioritization Matrix
```
              URGENT
                │
    ┌───────────┼───────────┐
    │           │           │
    │  DO NOW   │  SCHEDULE │
    │           │           │
I ──┼───────────┼───────────┼── NOT IMPORTANT
M   │           │           │
P   │  DELEGATE │  MINIMIZE │
O   │           │           │
R   └───────────┼───────────┘
T               │
A         NOT URGENT
N
T

Priority 1: CEO can't access email (Urgent + Important)
Priority 2: Server maintenance needed (Important, not urgent)
Priority 3: User wants software upgrade (Urgent to them, not important)
Priority 4: Optional training request (Neither urgent nor important)
```

### Communication Templates

#### Opening a Ticket
```
"Hello [Name], this is [Your Name] from IT Support. I understand 
you're experiencing [brief issue description]. I'm here to help 
resolve this for you. Can you tell me exactly what's happening?"
```

#### While Troubleshooting
```
"I'm checking [what you're doing] now. This will take about 
[time estimate]. While I'm doing this, can you tell me if 
[relevant question]?"
```

#### Resolution
```
"I've [what you did], and the issue should now be resolved. 
Can you please try [action] to confirm it's working on your end? 
... Great! Is there anything else I can help you with today?"
```

#### Escalation Needed
```
"I've identified the issue, but resolving it requires [reason - 
specialized tool/higher permissions/server access]. I'm going to 
escalate this to [team/person] who can [what they'll do]. They 
should contact you within [timeframe]. I'll add a note to the 
ticket so they have all the details. Is that okay?"
```

## 📝 Notable Scenarios & Solutions

### Scenario 1: Complex Email Issue
**Problem:** User could receive emails but not send. Multiple devices affected.

**Investigation:**
- Checked account settings - looked correct
- Tested on different device - same issue
- Reviewed recent changes - user changed password yesterday
- Checked outgoing server authentication - FOUND IT!

**Root Cause:** After password change, outgoing SMTP server still had old password cached.

**Solution:** Updated SMTP authentication password to match new account password on all devices.

**Time:** 15 minutes  
**Satisfaction:** 5/5  
**Learning:** Always ask "What changed recently?" - often reveals root cause.

---

### Scenario 2: Intermittent Network Issue
**Problem:** User experiencing random disconnections throughout the day.

**Investigation:**
- Continuous ping test showed packet loss
- Physical inspection revealed loose cable
- Replaced network cable
- Problem resolved

**Time:** 10 minutes  
**Satisfaction:** 5/5  
**Learning:** Don't overlook physical layer - "It's always the cable!"

---

### Scenario 3: Printer Nightmare
**Problem:** Entire department couldn't print to network printer.

**Investigation:**
- Individual computers could ping printer
- Printer showed ready status
- Print jobs stuck in queue
- Print spooler service stopped on print server

**Root Cause:** Print spooler service crashed due to corrupted print job.

**Solution:** 
1. Stopped print spooler service
2. Cleared print queue folder
3. Restarted print spooler
4. Tested printing - success

**Time:** 20 minutes  
**Satisfaction:** 4.8/5 (users very relieved!)  
**Learning:** Department-wide issues often indicate central service problem, not individual computer issues.
